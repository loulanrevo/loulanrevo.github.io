Blog Data Visualization: The Complete Guide to Mermaid Diagrams

This article demonstrates how to use Mermaid in your blog to create elegant data visualizations. All diagrams are written directly in Markdown code blocks, no external services needed.

Tech Stack & Plugins

Layer Tech/Plugin Description
Blog Framework Hexov8.1.1 Static blog generator
Theme Butterflyv5.5.4 Built-in Mermaid support, configure mermaid.code_write: true
Diagram Engine Mermaid JavaScript-based diagramming tool, supports flowcharts, sequence diagrams, Gantt charts, etc.
Syntax Conversion mermaid-convert.js Custom Hexo Filter plugin, automatically converts ````mermaidcode blocks to Butterfly's{% mermaid %}` tag
Style Theme darkmode.css Custom dark theme variables, ensures diagrams are clearly readable in dark mode

1. Flowchart

Flowcharts are the most commonly used diagram type, perfect for illustrating workflows, decision processes, and more.

Use Cases

  • Workflow documentation
  • Decision trees
  • Algorithm logic
  • System architecture

2. Sequence Diagram

Sequence diagrams show the order of interactions between objects, particularly suited for describing API calls, user actions, and similar scenarios.

Use Cases

  • Data statistics
  • Proportion analysis
  • Time allocation
  • Category display

4. Gantt Chart

Gantt charts are a project management staple, clearly showing task timelines.

Use Cases

  • Project progress management
  • Study plans
  • Release roadmaps
  • Task dependency relationships

5. Class Diagram

Class diagrams are used to illustrate code structure and object-oriented design.

Use Cases

  • Code documentation
  • Database design
  • API structure descriptions
  • System architecture

6. State Diagram

State diagrams show state changes and transition conditions of objects.

Use Cases

  • Process state management
  • Lifecycle descriptions
  • Business rules
  • State machine design

7. ER Diagram (Entity Relationship)

ER diagrams illustrate relationships between database entities.

Use Cases

  • Database design
  • Data model descriptions
  • API relationship documentation
  • System architecture

8. Mindmap

Mindmaps are great for presenting hierarchical knowledge structures.

Use Cases

  • Knowledge organization
  • Idea structuring
  • Project planning
  • Learning paths

Usage Tips

  1. Keep it simple: Each diagram should convey one core idea
  2. Choose wisely: Select the appropriate diagram type based on your data
  3. Mind readability: Avoid overly complex nested structures
  4. Add explanations: Pair diagrams with text descriptions
  5. Test rendering: Preview before publishing to ensure proper display
